# TriageAI React Frontend — Backend Contract

This React app is backend-connected by default. The backend/database is the source of truth for real assessments, dashboard summaries, audit events, clinician review, and official PDF reports.

## Clinical Scope

- Model display name: `LightGBM V2 Weight + Threshold`
- Model version from `/ready`: `lightgbm_v2_weight_threshold_esi345`
- Model source from `/ready`: `final_registry`
- Model registry path: `model_registry/esi_345_lightgbm_v2`
- Clinical wording: decision-support, ESI care routing, clinician review, not diagnosis, does not replace clinician judgment

The model supports ESI 3/4/5 prediction. Safety rules and clinician review can escalate the final routing decision to ESI 1/2 when high-risk criteria are present.

## Backend Endpoints

```http
GET  /health
GET  /ready
GET  /dashboard/summary
GET  /assessments
GET  /assessments/{assessment_id}
GET  /assessments/{assessment_id}/audit
GET  /assessments/{assessment_id}/report/pdf
POST /predict
POST /clinician-review
POST /reports/generate
GET  /reports/{report_id}/download
```

## Report Strategy

The official report is generated by the backend ReportLab service. React must call:

```http
GET /assessments/{assessment_id}/report/pdf
```

The legacy browser-side `reportPdf.ts` helper is retained only for mock-mode experiments and must not be used as the official report source.

## Mock Data Policy

`mockData.ts` and `mockApi.ts` are development/test fixtures only. Enable synthetic data explicitly with:

```env
VITE_USE_MOCK_API=true
```

Production workflows must leave `VITE_USE_MOCK_API=false` and use `VITE_API_BASE_URL=http://localhost:8001` for local FastAPI.

## Storage and Deployment

Local development uses SQLite:

```env
DATABASE_URL=sqlite:///./triageai.db
```

Azure or other production deployments should use PostgreSQL. Do not track generated files such as `node_modules`, `dist`, `*.tsbuildinfo`, local SQLite databases, or generated PDFs.
